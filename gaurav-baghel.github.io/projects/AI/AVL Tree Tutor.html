Abstract

What was our Aim?

This paper describes our new visual understanding based intelligent-state-feedback approach, for teaching the insertion algorithm for the AVL trees. Our aim is to supplement basic class lectures with an interactive intelligent tutoring system that integrates our proposed approach for better understanding.


A more extensive explanation of the developed tutor is available here in this video.

Research

Why Data Structures?

It is more probable that a motivated student is a productive learner. Interesting assignments encourage learning by actively involving students in the material. Active learning for students is very essential in an introductory course on data structures where students learn the basics of programming. It can be both challenging and hard to learn for any individual student.

Why AVL Trees?

Many approaches have been tried but not with very successful implementation outcomes. One excellent suggestion was to use visual programming for teaching the concept of Binary Trees. AVL (Adelson-Velskii and Landis) trees in Data Structures is one such topic where many students face difficulty in understanding the integral rotations for balancing.

The Approach.

Past approaches for teaching AVL tree using an online tutor include resubmissions to restart the exercise with new random data, using animations and visualizations to show how the rules are applied. It has been seen that even with the aid of these approaches, students still have difficulty with creating an AVL tree from a series of input values. This is due to the fact that almost all of these approaches lack a proper feedback system that is required for essential persistent learning.

Studies have indicated that immediate feedback method actively engages learners in the discovery process and that this engagement promotes correction and the retention of initially inaccurate response strategies.

Solution

What is our tutor?

AVL-Tree tutor, speaking holistically, is a single page web application which consists of two basic parts. First part is a tutorial section specifying main concepts related to insertion in an AVL tree and the second part is the tutor section which students can use to solve example problems.

Tools.

We developed this Intelligent Tutoring System using Cognitive Tutor Authoring Tools (CTAT). CTAT is free for research tool developed by Carnegie Melon University for Easy demonstration and development of Learning tools. This was developed to significantly reduce the amount of time needed to create a standalone web-tutoring-application.

How student a should use it?

The Tutor screen consists of a set of numbers that the student is required to fill in the empty tree. This is done by inputting the appropriate value of the nodes by filling in the textboxes inside each node. It imposes ordering constraints on problem-solving with multiple paths leading to the correct answer. When the student makes a mistake at any step, that step (node) is highlighted by red color and the tutor won't move to the next one until it's corrected.

When a student requests for a hint, the first hint from the relevant link is displayed into the white hint box. A link can have multiple levels, where each successive request by the student displays the next level of hint. The last or lowest level of hint tells the student where to exactly insert the element in the tree.

When all elements are inserted, the student can let the tutor know by clicking on the Done button.

Background Implementation.

CTAT basically utilizes two basic components to model a tutor application – An HTML Interface and A state-behavior graph. Using CTAT we made the behavior graph to model different problems of AVL trees. This detailed graph can be used to control whether or not to show hints and other feedback to the student at any point of time or step. Basically, this graph is a set of states (Incorrect or correct) that the tutor traverses through in order to guide a student through the correct path with proper feedback at each step.

Reflections

Future Scope.

For future work, we aim to implement our tutor in the running classroom course of data structures and collect data by taking 2 tests of the enrolled students after completing classroom lectures - one before using this tutor and another after using it for measuring the changes in student scores. The change in the scores of students would be a reflector of the effectiveness of this tutor. Since this tutor would be an addition to the existent classroom teaching which is almost entirely theoretical, we presume that this little addition would cater for the practical requirements of the student learning of the topic and enhance the overall learning experience of the classroom.
